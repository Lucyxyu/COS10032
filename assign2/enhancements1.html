Bootstrap

Bootstrap is a very popular open source CSS development framework for websites. Bootstrap has many libraries and we used the CSS one for responsive design. This went beyond the requirements of the assignment as this required a lot of research and trial and error to successfully implement on the page.

The Bootstrap techniques used were from two main sources:

  • This Youtube tutorial
  • The Bootstrap official website

To add the Bootstrap CSS library, put the following code in the head tag:
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-QWTKZyjpPEjISv5WaRU9OFeRpok6YctnYmDr5pNlyT2bRjXh0JMhjY6hW+ALEwIH" crossorigin="anonymous">

  • The use of Bootstrap is evident on the topic page on the aside element when the page width is sized up and down. Click here. The bootstrap class used for this element is:
    <aside class="col-md-4 col-12"></aside>
    This translates to "when the screen is less than medium width, make this element span across the whole page inside of just on the right".
  • Bootstrap can also been seen on the table element on the topic page, where the rows will highlight upon mouse hover. Click here.

Image map

An image map has been used to better assist the user in understanding of the topic. Click here.

This is an extension from the assignment as it is an innovative addition to the standard img element requirement. Rather than nesting an image in a single anchor tag, different clickable areas of the image link the user to different actions. The clickable areas are defined by co-ordinates on the image. The following code was used:
<img id="fig2-image" src="../images/JS-CSS-HTML.png" alt="3 layer standard web technologies" usemap="#weblevels">
<map name="weblevels">
<area shape="poly" coords="12,26,154,11,298,26,298,64,156,82,12,63" alt="javascript" href="topic2.html">
<area shape="poly" coords="13,75,12,113,156,131,299,113,298,75,249,70,156,83,63,71" alt="css" href="topic3.html">
<area shape="poly" coords="12,125,12,162,155,181,298,163,298,126,247,119,155,131,68,119" alt="html" href="topic4.html"> </map>

The research and resources required to implement the image map include:

  • w3schools tutorial
  • Image map co-ordinate generator

It was discovered upon implementation that image maps do not work well with dynamic resizable images therefore the image used is of a fixed size.

Favicon

A favicon is a small icon found next to anything that identifies the website. It is ussually visible in the title bar of a web browser, bookmarks, search history or search bar. Although a favicon is only 16 x 16 pixels in size, we felt it was an important detail to include due to the following:

  • A visible favicon helps users save time by allowing them to quickly identify the website, thus increasing user engagement
  • Websites without a favicon may be looked at unfavorably by certain browsers for search ranking purposes
  • Some of the most famous websites in the world have highly identifiable Favicons (google, facebook etc)
  • Booksmarks without favicons are not user friendly

To create a favicon, go to favicon.io and use the generator. Once downloaded, add it to the HTML head tag of an element with the following code:
<link rel="icon" type="image/x-icon" href="pathhere">

Tooltips

Tooltips have been added to the email signature images in the footer of the website upon hover as per below.

These enhance the website as they give the user a prompt and a call to action. The tooltips only appear when necessary, keeping the interface clean and simple.

The code required for the tooltips are:
Adding a span class in the HTML with tool tip text
<span class="tooltiptext">Email Ravinda!</span>
Adding CSS properties to only unhide the text upon hover
.email-individual .tooltiptext {
visibility: hidden;
position: absolute;
z-index: 1;
bottom: 120%;
right: 0%;
white-space: nowrap;
opacity: 0;
transition: opacity 1s;
font-family: Arial, sans-serif;
font-size: 0.8em;
background-color: rgb(245, 243, 243);
padding: 0.5em;
border: 1px solid;
border-radius: 1em;
}

.email-individual:hover .tooltiptext {
visibility: visible;
opacity: 1;
}

The technique for making these were sourced from the following:

  • w3schools tutorial
  • FreeCodeCamp forum post

Flexbox

Due to the vision of what we wanted the header to look like, it was necessary to extend beyond the syllabus material, hence flexbox was used for the heading. Flexbox makes it much easier to align items and is also responsive to viewport resizing.

Try to resize this page while looking at the header. At narrow widths, the h1 title text will disapear, however the navigation links are still visible and will move left to occupy the full row.

The CSS code used to made a flexbox is:
display: flex;
flex-direction: row;
justify-content:space-between;
align-items: center;
 This lets the browser know the header element is a flex box in a horizontal direction and that content should be aligned evenly across the top.
